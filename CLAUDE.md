### CSS Architecture Rules (funnel engine)

**Full-width elements inside `.paywall`:**
- Do NOT rely on `align-items: stretch` cascade to make paywall children full-width. The `.content` base class sets `align-items: center`, and cascade order isn't always reliable in practice.
- Instead, use **explicit `width: 100%; box-sizing: border-box`** on any element that must fill the paywall width.
- The rule `.paywall > * { width: 100%; box-sizing: border-box; }` in `engine/styles.css` covers all direct children — maintain this.

**Dead CSS cleanup on migration:**
- When migrating screens/components to the shared engine, search for and remove any old CSS blocks that generated class names the engine no longer produces (e.g. old `.before-after__*` grid layout). Dead CSS silently conflicts with new layouts.
- After migration, grep for class names in the old CSS and verify they're still generated by `engine/app.js`.

**Paywall data source of truth:**
- The canonical paywall data (EUR pricing, correct disclaimers) lives in `funnel/funnels/v2/screens.json`.
- When creating new funnels based on v2, copy screens.json from there — not from any archived `funnel-v2/` directory or pre-Issue-#17 backup files.
